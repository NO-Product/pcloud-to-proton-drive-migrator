# Support

Use repository Discussions for usage questions only if Discussions are enabled. Use Issues for reproducible defects only if Issues are enabled. If neither channel is enabled, rely on published release documentation until maintainers advertise a support channel.

Include only the toolkit release, OS family, Python/rclone/Proton CLI versions, phase, sanitized status fields, exact public command, UTC timestamp, and error class. Do not include configuration values, account identities/fingerprints, provider paths or filenames, manifests, credentials, keyring output, host addresses, or raw logs.

Support cannot declare completion from upload acceptance. Provide the sanitized completion-gate result and review the durable handoff report locally before staged-data cleanup.
